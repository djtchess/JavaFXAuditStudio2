Tu extrais uniquement ce qui est demandé.
Tu n’inventes rien.
Tu ne complètes rien.

Tu ne tronques rien.

# REGLES GLOBALES

* chaque élément extrait doit correspondre exactement à un texte visible dans le code
* comparaison et déduplication sensibles à la casse
* conserver l’ordre de première apparition dans le code
* ne jamais sortir deux fois exactement la même ligne dans une même section
* si une information n’existe pas, retourner une liste vide pour la section
* ignorer les imports
* ignorer les appels de méthodes comme source d’extraction
* ignorer les noms seulement utilisés sans déclaration explicite
* ignorer les noms déduits ou reconstruits
* exclure les constructeurs de METHODES
* exclure les paramètres de constructeurs de PARAMETRES_METHODES
* exclure les paramètres de constructeurs de VARIABLES_LOCALES_METHODES

Règles :

# REGLE DE DEDUPLICATION ABSOLUE

Pour chaque section :

* construire d’abord une liste brute de candidats
* construire ensuite une liste finale vide
* parcourir la liste brute dans l’ordre
* pour chaque candidat :

  * si ce candidat est déjà présent exactement dans la liste finale, l’ignorer
  * sinon l’ajouter
* la sortie finale doit être produite uniquement à partir de la liste finale
* il est interdit de produire une section à partir de la liste brute
* aucune ligne identique ne doit apparaître deux fois dans une même section

# CONTROLE FINAL BLOQUANT DE DEDUPLICATION

Avant de rendre la réponse :

* relire chaque section ligne par ligne
* si une ligne est déjà apparue plus haut dans la même section, la supprimer
* conserver uniquement la première occurrence exacte de chaque ligne
* seulement ensuite produire la réponse finale


# DEFINITIONS PAR SECTION

## PACKAGE_INTERNE

Source :

* uniquement la ligne package

Sortie :

* la valeur exacte de la ligne package

## CLASSES

Source :

* déclarations de classes, interfaces, enums, records et classes internes
* types visibles dans les déclarations de champs
* types visibles dans les champs annotés avec @Autowired
* types visibles dans les signatures de méthodes
* types visibles dans les paramètres
* types visibles dans les variables locales
* types visibles dans les instanciations explicites

Inclure :

* noms de classes déclarées
* noms de classes référencées comme types
* types de champs @Autowired

Exclure :

* imports
* annotations
* noms de variables
* noms de méthodes
* mots-clés Java
* noms reconstruits

Règles :

* si un identifiant est le type d’un champ, l’extraire dans CLASSES
* si un identifiant est le type d’un champ annoté avec @Autowired, l’extraire dans CLASSES
* ne jamais extraire le nom du champ à la place du type

## METHODES

Source :

* uniquement les déclarations réelles de méthodes

Règles :

* extraire uniquement le nom exact de la méthode
* exclure les constructeurs
* ne pas inclure signature, paramètres, type de retour, annotations

## PARAMETRES_METHODES

Source :

* uniquement les paramètres déclarés dans les signatures réelles des méthodes

Inclure :

* paramètres visibles sur une ou plusieurs lignes

Exclure :

* paramètres de constructeurs
* variables locales
* attributs
* constantes
* types
* annotations

Règles :

* extraire uniquement le nom exact complet
* si le nom complet n’est pas lisible explicitement, ne rien extraire
* supprimer tout nom interdit
* dédupliquer strictement en gardant la première occurrence exacte

## VARIABLES_LOCALES_METHODES

Source :

* uniquement les variables locales explicitement déclarées dans le corps des méthodes

Inclure :

* variables locales classiques
* variables de for
* variables de for-each
* variables de try-with-resources
* variables de catch
* variables de déclarations multiples
* variables déclarées sur plusieurs lignes si le nom est explicitement visible

Exclure :

* paramètres de méthodes
* paramètres de constructeurs
* attributs
* constantes
* noms seulement utilisés
* types
* annotations

Règles :

* extraire uniquement le nom exact complet
* ne pas inclure le type
* ne pas inclure l’initialisation
* dans `int a = 1, b = 2;`, extraire `a` et `b`
* si le nom complet n’est pas lisible explicitement, ne rien extraire
* supprimer tout nom interdit
* dédupliquer strictement en gardant la première occurrence exacte

## ATTRIBUTS

Source :

* déclarations réelles de champs
* accès attributaires explicites visibles

Inclure :

* noms de champs déclarés
* noms visibles dans `this.nomAttribut`
* noms visibles dans `objet.nomAttribut`
* champs annotés @FXML, @Autowired, etc.

Exclure :

* noms de méthodes
* variables locales
* paramètres
* noms déduits depuis getter/setter

Règles :

* extraire uniquement le nom exact de l’attribut
* ne jamais extraire un nom seulement supposé être un attribut

## CONSTANTES

Source :

* uniquement les champs déclarés avec static final

Règles :

* extraire uniquement le nom exact de la constante

## CHAINES_CARACTERES

Source :

* uniquement les chaînes littérales visibles

Règles :

* conserver exactement la valeur visible
* dans une concaténation, garder chaque morceau littéral séparé

## CLASSES_INJECTEES

Source :

* uniquement les champs annotés avec @Autowired

Règles :

* extraire uniquement le type exact injecté
* ne jamais extraire le nom du champ
* ne jamais extraire l’annotation
* si le type est générique, conserver uniquement le type principal visible

# REGLES TRANSVERSES

* un champ @Autowired peut contribuer à plusieurs sections :

  * ATTRIBUTS : nom du champ
  * CLASSES : type du champ
  * CLASSES_INJECTEES : type du champ
* une constante static final va dans CONSTANTES
* une variable locale ne doit jamais apparaître dans ATTRIBUTS
* un paramètre de méthode ne doit jamais apparaître dans VARIABLES_LOCALES_METHODES
* un identifiant partiel ou tronqué est interdit en sortie

# ALGORITHME OBLIGATOIRE

1. lire TYPE_SORTIE
2. construire la liste effective d’exclusion :

   * utiliser NOMS_A_EXCLURE_DES_VARIABLES_DE_METHODES si elle est renseignée
   * sinon utiliser :

     * newValue
     * oldValue
     * value
     * newSelection
     * oldSelection
3. extraire les candidats bruts de la ou des sections demandées
4. supprimer tout candidat non textuellement visible tel quel dans le code
5. pour PARAMETRES_METHODES et VARIABLES_LOCALES_METHODES :

   * supprimer les noms interdits
   * construire une liste finale vide
   * parcourir les candidats dans l’ordre d’apparition
   * ajouter un candidat seulement s’il n’est pas déjà présent exactement dans la liste finale
6. pour les autres sections :

   * dédupliquer en conservant la première occurrence exacte
7. avant de répondre :

   * vérifier que PARAMETRES_METHODES ne contient aucun nom interdit
   * vérifier que VARIABLES_LOCALES_METHODES ne contient aucun nom interdit
   * vérifier qu’aucune section ne contient de doublon exact
   * vérifier qu’aucun identifiant n’est tronqué
8. produire la sortie finale
